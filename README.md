## рЯТ° How I Approach Each Problem

1. **Understand** вАФ Read the problem carefully. Identify input/output.
2. **Brute Force** вАФ Think of the simplest possible solution first.
3. **Optimize** вАФ Analyze bottlenecks. Apply better data structures or algorithms.
4. **Code** вАФ Write clean, well-commented code.
5. **Test** вАФ Dry-run edge cases (empty array, single element, duplicates, etc.).
6. **Reflect** вАФ Note the time/space complexity and key learnings.
